# Инструкция по деплою Telegram-бота на Bothost.ru

Эта инструкция поможет вам развернуть вашего Telegram-бота на платформе Bothost.ru.

## 1. Подготовка кода бота

Убедитесь, что ваш код бота готов к деплою. В вашем проекте должны быть следующие файлы:

- `main.py`: Основной файл с логикой бота.
- `requirements.txt`: Список Python-зависимостей.
- `Dockerfile`: Файл для сборки Docker-образа (если вы хотите использовать собственный Dockerfile).

## 2. Создание репозитория Git

Bothost.ru поддерживает деплой из Git-репозиториев (например, GitHub, GitLab, Bitbucket). Если у вас еще нет репозитория для вашего бота, создайте его и загрузите туда все файлы проекта.

Пример структуры репозитория:

```
my_telegram_bot/
├── main.py
├── requirements.txt
└── Dockerfile
```

## 3. Настройка переменных окружения

Ваш бот использует переменные окружения для хранения конфиденциальных данных, таких как токен бота и ID модераторов. Вам нужно будет установить их на Bothost.ru.

### `BOT_TOKEN`

Это токен вашего Telegram-бота, который вы получили от BotFather. **Никогда не храните этот токен непосредственно в коде или в открытом доступе в вашем репозитории Git.**

### `MODERATOR_IDS`

Это список числовых идентификаторов Telegram-пользователей, которые будут выступать в роли модераторов. Разделите их запятыми без пробелов. Например: `123456789,987654321`.

## 4. Деплой на Bothost.ru

1.  **Войдите в панель управления Bothost.ru.**
2.  **Создайте нового бота или выберите существующий проект.**
3.  **Укажите Git URL репозитория:** В поле `Git URL репозитория` вставьте ссылку на ваш репозиторий Git (например, `https://github.com/username/my_telegram_bot.git`).
4.  **Укажите ветку:** В поле `Ветка` укажите ветку, из которой будет производиться деплой (по умолчанию `main`).
5.  **Добавьте переменные окружения:**
    *   Найдите раздел `Переменные окружения`.
    *   Нажмите `+ Добавить переменную окружения`.
    *   Для `BOT_TOKEN`: в поле `Ключ` введите `BOT_TOKEN`, в поле `Значение` вставьте ваш токен бота.
    *   Для `MODERATOR_IDS`: в поле `Ключ` введите `MODERATOR_IDS`, в поле `Значение` вставьте список ID модераторов через запятую (например, `123456789,987654321`).
    *   Повторите для всех необходимых переменных.
6.  **Укажите главный файл (точка входа):** В поле `Главный файл` укажите `main.py`.
7.  **Использование собственного Dockerfile (опционально):** Если вы хотите, чтобы Bothost.ru использовал ваш `Dockerfile` для сборки, установите флажок `Использовать собственный Dockerfile`.
8.  **Нажмите кнопку "Создать бота" или "Обновить бота"** (в зависимости от того, создаете вы нового бота или обновляете существующего).

Платформа Bothost.ru автоматически загрузит ваш код, установит зависимости и запустит бота.

## 5. Проверка работы бота

После деплоя проверьте работу бота в Telegram. Отправьте ему команду `/start` и убедитесь, что он отвечает корректно, отображая информацию о тарифах и кнопку "Наш ТГК". Также проверьте функционал модерации, отправив сообщения от обычного пользователя и пытаясь взять их в работу модератором.
